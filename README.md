# M5AtomPoeArtNet2DMX

Firmware for an M5Stack Atom Lite, Atomic PoE Base, and M5Stack DMX Unit. The device bridges Art-Net and DMX512, stores settings in LittleFS, serves a web management UI, supports JSON backup and restore, and advertises `m5atompoe-artnet2dmx.local` through mDNS.

## Hardware

| Part | Function |
| --- | --- |
| M5Stack Atom Lite | ESP32 controller |
| Atomic PoE Base | W5500 SPI Ethernet with PoE power |
| M5Stack DMX Unit | DMX512 UART interface on Grove Port C |

| Signal | GPIO |
| --- | --- |
| W5500 SCK | 22 |
| W5500 MISO | 23 |
| W5500 MOSI | 33 |
| W5500 CS | 19 |
| DMX TX | 26 |
| DMX RX | 32 |
| Atom Lite button | 39 |

## Network Behavior

The Ethernet interface uses DHCP or a stored static IPv4 configuration. WiFi station mode is used as the next network path after Ethernet startup completes and stored WiFi credentials exist. The Atom Lite front button starts the setup access point during startup or runtime.

The setup access point uses the configured hostname as its SSID and serves the captive portal at `http://192.168.4.1/`.

The web UI is available on the active network interface at:

- `http://m5atompoe-artnet2dmx.local/`
- `http://<device-ip>/`

## Art-Net

The Art-Net node listens on UDP port `6454`. ArtPoll discovery replies include the configured short name, long name, Net, Subnet, Universe, IP address, and DMX port metadata.

Art-Net addressing uses:

- Net: `0` to `127`
- Subnet: `0` to `15`
- Universe: `0` to `15`
- Port address: `(Net << 8) | (Subnet << 4) | Universe`

Unicast transmission uses the configured target IPv4 address. Broadcast transmission uses the active interface broadcast address.

## DMX Modes

`Art-Net to DMX` maps ArtDmx slot `1` to the configured DMX start address and sends a continuous DMX512 output stream.

`DMX to Art-Net` reads DMX frames from the DMX Unit and sends ArtDmx packets beginning at the configured DMX start address. The transmit target follows the Broadcast or Unicast setting.

## Settings

Settings are stored in `/config.json` on LittleFS:

- Hostname
- WiFi SSID and password
- WiFi SSID scan results from the active radio environment
- Ethernet DHCP or static IPv4 configuration
- Art-Net short name and long name
- Art-Net Net, Subnet, and Universe
- DMX start address
- DMX mode
- Broadcast or Unicast transmit mode
- Unicast target IP address

The web UI provides:

- Live network and packet status
- Settings editing
- JSON backup download
- JSON restore upload
- Reboot command

## Build And Upload

The PlatformIO environment pins Espressif 32 platform `6.6.0` with Arduino ESP32 `2.0.14` for the DMX driver dependency.

Install PlatformIO, connect the Atom Lite over USB, then run:

```bash
pio run
pio run --target upload
pio device monitor
```

LittleFS stores runtime settings generated by the firmware. A filesystem image upload is optional for this project.
